## Why

Locator Drift can validate the correct payment candidate but still fail rerun proof if the executable patch line comes from the AI response instead of SpecHeal's controlled patch builder. The observed failure created a locator variable and never clicked it, so the test waited for `Payment Success` without performing payment.

## What Changes

- Make controlled HEAL patch application generate the executable Playwright click line from the validated selector.
- Treat OpenAI patch text as non-authoritative; OpenAI may choose a candidate, but SpecHeal owns the runtime patch shape.
- Repair the controlled ShopFlow action region before patching so a previous malformed runtime patch does not poison later runs.
- Clarify proof display when rerun blocks patch preview.

## Capabilities

### Modified Capabilities
- `specheal-recovery`: Safe HEAL rerun proof now requires a canonical click patch generated by SpecHeal, not an arbitrary AI-provided line.

## Impact

- Affected code: controlled Playwright patch application, rerun proof reporting, MVP verification checks.
- Affected systems: no database migration, no Jira behavior change, no Kubernetes manifest change.
